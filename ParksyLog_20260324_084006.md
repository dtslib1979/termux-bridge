그러니까 Termux가 GitHub "같은" 건 아니고, GitHub에 접근하게 해주는 물리적 인터페이스야. Termux 없으면 폰에서 이 생태계 전체가 손 닿지 않는 거지.

tmux = Terminal Multiplexer (터미널 다중화기)
ls = list sessions (세션 목록 보기)
new -s claude-main = new session, 이름은 claude-main

git push origin main = 원격(GitHub) main 브랜치에 업로드
git diff --cached --stat = 스테이징된 변경사항 통계

인터넷은 "어떻게 전달하느냐"가 아니라 "어떤 규칙으로 대화하느냐"야:

전달 수단 (물리적)	인터넷이냐?
구리 전화선 (옛날 ADSL)	✅ 위에서 인터넷 프로토콜 돌리면 인터넷

위성 (스타링크)	✅ 위에서 인터넷 프로토콜 돌리면 인터넷
FM 라디오 전파	❌ 인터넷 프로토콜 안 돌림
TV 방송 전파	❌ 인터넷 프로토콜 안 돌림
전파 자체가 인터넷이 아니야. 전파든 케이블이든 위성이든 — 그 위에 TCP/IP라는 약속을 올려서 돌리면 그게 인터넷이야.

군사 기원 이야기:

1969년: 미국 국방부 ARPANET

  → 중앙 서버 없이 분산 네트워크 설계
  → 한 노드 죽어도 다른 경로로 우회
  → 이게 인터넷의 원형

계층 구조로 보면:

Layer 1: 물리 (전선, 전파, 광케이블) → 신호를 전달
Layer 2: 연결 (WiFi, LTE, 이더넷) → 기기끼리 연결
Layer 3: 네트워크 (IP 주소) → 목적지 찾기
Layer 4: 전송 (TCP/UDP) → 데이터 안전하게 전달
Layer 5+: 응용 (HTTP, SSH, FTP) → 실제 서비스

LTE 전파는 택배 트럭이야. 인터넷 프로토콜은 운송장이야. 트럭이 뭐든 — 오토바이든 비행기든 — 운송장 규칙이 같으면 택배는 도착해. 그게 인터넷이야.

그래서 WiFi에서 LTE로 바뀌어도 인터넷은 되는 거야. 트럭만 바뀐 거지 운송장(프로토콜)은 같으니까. 다만 트럭 갈아타는 순간 SSH 연결이 잠깐 끊기는 거고, tmux가 그걸 커버하는 거지.

```
TCP (1974) = 확인하고 보내는 캐치볼

    → HTTP (1991) = TCP 위에서 "웹페이지 주세요" 주문
      → REST API (2000) = HTTP로 "데이터 CRUD" 체계화
        → WebSocket (2011) = HTTP로 시작하고 전화 통화로 전환
          → OAuth 2.0 (2012) = 비번 대신 위임장으로 인증
            → SSH (1995) = TCP 위에 암호화 터널
```
