## 3) PC측 작업 사실 (이미 한 일)
- `onnxruntime.quantization.quantize_static(QuantFormat.QDQ, QUInt8/QInt8, per_channel=False)`
- 5쌍 중 3쌍 INT8 변환 성공 (encoder×2, vits×1)
- decoder 2쌍은 ONNX export 자체 버그(`Rank of input 'data' needs to be equal to rank of input 'indices'`)로 양자화 불가 → fp32 inflated 그대로 수송
- 박씨 dataset 20개 + 박씨 wav 30개로 calibration → **이전 폰 단독 동적 양자화(cos 0.988)보다 정확도 우월할 가능성 높음** (정적 + 박씨 본인 분포 사용)
