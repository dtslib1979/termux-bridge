# AI 에이전트 + ADB + Tailscale 폰 원격제어 — 포지셔닝 정리 (2026-03-26)

## 선구자 검증

같은 구조로 이미 검증된 사례들:

- **AI 에이전트(Claude Code, Cursor 등) + ADB + Tailscale 안드로이드 원격제어** 가이드가 GitHub에 존재
- **카페 노트북 + LTE 폰** 조합으로도 항상 작동 (Tailscale이 NAT 뚫어줌)
- **집 서버 cron** → 폰 자동 제어도 가능
- ADB TCP 상시 켜도 슬립 안 막음, 배터리 영향 거의 없음 (simonsafar.com 테스트 결과)

방향이 맞다. 선구자들이 기술적으로 검증해놓은 길.

## 박씨가 특이한 지점

ADB 쓰는 사람 많다.
Tailscale 쓰는 사람 많다.
SSH 쓰는 사람 많다.

**이걸 전부 엮어서 AI가 폰을 원격 제어하는 시스템으로 만드는 사람은 거의 없다.**

존재하는 가이드들은 전부 개발자가 쓴 것.

박씨가 다른 점:

> **비개발자가 3년간 AI 협업만으로**
> Tailscale + SSH + BIOS 상시부팅 + WSL + Termux + ADB를
> 하나하나 쌓아서 여기까지 왔다.

개발자한테 각각의 기술은 쉽다.
**이 조합을 설계하고 유지하는 건 아키텍처 레벨 작업이다.**

## 오늘 삽질의 의미

Android 15가 보안 강화하면서 무선 ADB 페어링을 막았다.
개발자들도 똑같이 고생한다.
박씨 잘못이 아니다.

USB 한 번 꽂으면 끝나는 문제 — 그 다음은 영구 무선.

## 콘텐츠 각도

이 삽질 자체가 콘텐츠다:

- "비개발자가 AI랑 같이 이 시스템 만드는 과정"
- 개발자들도 똑같이 막히는 Android 15 벽
- 해결 방법 (USB 한 번 → tcpip 5555 → 영구 무선)

**개발자들한테는 당연한 기술이지만, 비개발자가 설계하고 완성했다는 서사가 차별점이다.**
