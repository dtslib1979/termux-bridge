## 1장. 세계관 — 내가 서 있는 곳

### 1-1. 층위 구조 (Layer)

Termux를 켰을 때 어디에 있는지를 모르면 아무것도 못 한다.
총 다섯 층이 있다.

```
Layer 4 │ SSH + Tailscale    ← 집 PC를 원격 제어하는 터널
Layer 3 │ Claude Code        ← AI 에이전트가 작업 실행
Layer 2 │ tmux               ← 창 유지, 세션 보존
Layer 1 │ bash 쉘            ← 리눅스 명령어 공간
Layer 0 │ Termux 앱          ← 안드로이드 위에서 돌아가는 터미널
```

**내가 지금 어느 층에 있는지 어떻게 아는가:**

| 화면 표시 | 위치 |
|----------|------|
| `$` 또는 `%` | bash 쉘 (Layer 1) |
| `❯` | Claude Code 안 (Layer 3) |
| `tmux` 상태바 보임 | tmux 안 (Layer 2) |
| `dtsli@PC이름` | 집 PC에 SSH로 접속된 상태 (Layer 4) |

### 1-2. 핸드폰 vs PC — 역할 분담

2026년 3월, 역할이 바뀌었다.

| 항목 | Before | After (지금) |
|------|--------|-------------|
| 메인 기기 | 핸드폰 Termux | 집 PC WSL2 |
| 보조 기기 | PC (서버) | 핸드폰 (SSH 클라이언트) |
| 브라우저 자동화 | headless 우회, 각종 삽질 | PC Chrome 직접 |
| 배치 작업 | 핸드폰 한 세션 | tmux 던져놓고 퇴근 |

**핵심:** 핸드폰은 "집 PC에 말을 거는 창구"가 됐다.

### 1-3. 삼중 보험 구조

```
1차: PC WSL2 Claude Code    ← 메인 작업
2차: Windows Claude Code    ← WSL이 못하는 것 + 비상복구
3차: Termux (핸드폰)        ← 둘 다 죽었을 때 SSH 재접속
```

**왜 이 구조인가:**
자기가 자기를 수술할 수 없다.
WSL이 죽으면 WSL 안의 Claude Code도 죽는다.
그래서 두 환경이 서로를 살려준다.

```
WSL Claude Code          Windows Claude Code
(리눅스 환경)            (PowerShell 환경)
    │                         │
못 건드리는 것:          못 건드리는 것:
- 레지스트리              - tmux 세션
- Windows 서비스          - SSH 데몬
- Task Scheduler          - 리눅스 패키지
    │                         │
    └────── 서로가 서로를 살린다 ──────┘
```

## 2장. Termux 기초 조작

### 2-1. 물리 버튼이 키보드다

화면 키보드가 `Ctrl`, `Esc`, `Tab`을 못 낼 때:

| 조합 | 역할 |
|------|------|
| 볼륨 다운 (물리) + C (화면) | Ctrl+C — 실행 중단 |
| 볼륨 다운 + Z | Ctrl+Z — 프로세스 백그라운드 |
| 볼륨 다운 + D | Ctrl+D — 세션 EOF/종료 |
| 볼륨 다운 + L | 화면 클리어 |
| 볼륨 다운 + [ | Esc |
| 볼륨 다운 + T | Tab 자동완성 |

> **주의:** 볼륨 다운은 폰 옆에 있는 물리 버튼이다. 화면 키보드 아니다.

### 2-2. 세션 관리

**세션 열기:**
- 화면 왼쪽 가장자리에서 오른쪽으로 스와이프 → 세션 목록 나옴
- `NEW SESSION` 탭하면 새 세션 생성

**세션 종료:**
- `exit` 입력 → 현재 쉘 세션 종료
- 세션 길게 누르기 → KILL

**완전히 먹통일 때:**
- Termux 앱을 최근 앱 목록에서 밀어서 강제종료
- 다시 열면 새 세션 시작 (tmux 썼으면 `tmux attach`로 복구 가능)

### 2-3. Claude Code 층에서 bash 층으로

Claude Code (`❯`) 안에 있을 때 bash(`$`)로 나오는 법:

```bash
# ❯ 프롬프트에서:
exit       # bash 쉘로 복귀

# 또는 볼륨다운 + Z → bash 백그라운드 suspend
# fg 입력하면 Claude Code 다시 돌아옴
```

**Claude Code가 먹통일 때:**
```bash
# bash $ 에서:
fg         # Claude Code 살아있으면 복귀
# 또는
command claude   # alias 무시하고 원본 Claude Code 실행
```

## 3장. 집 PC를 서버로 만들기

### 3-1. 왜 서버화가 필요한가

실제 사고 경위:
```
외출 3일차 → Windows 자동 업데이트 발동
          → 강제 재부팅
          → SSH 끊김
          → WSL 세션 전부 사망
          → 3일치 원격 작업 환경 날아감
```

집 PC가 서버이자 사무실이자 생산공장이다.
24시간 원격 제어해야 하는데 꺼지면 안 된다.

### 3-2. 서버화 4단계

#### 단계 1: Windows 자동 재부팅 차단

WSL에서 직접:
```powershell
# WSL bash에서 powershell.exe 호출
powershell.exe -Command "New-Item -Path 'HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU' -Force"
powershell.exe -Command "Set-ItemProperty -Path 'HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU' -Name 'NoAutoRebootWithLoggedOnUsers' -Value 1 -Type DWord -Force"
```

> **주의:** 레지스트리 수정은 관리자 권한 필요. Windows Claude Code(관리자 PowerShell)에서 실행해야 한다.

#### 단계 2: WSL 자동 종료 방지

`/mnt/c/Users/dtsli/.wslconfig` 생성:
```ini
[wsl2]
vmIdleTimeout=-1
memory=7GB
```

`vmIdleTimeout=-1` = Windows idle 시 WSL VM 자동 종료 방지.
이거 없으면 PC 안 쓸 때 WSL이 자동으로 죽는다.

#### 단계 3: Windows 자동 로그인

재부팅 후 PIN 화면에서 막히면 원격 접속 불가.

**해결:** Sysinternals Autologon (Microsoft 공식 도구)
- 비밀번호를 LSA 암호화로 저장 → PIN 있어도 자동 로그인

> `netplwiz`, 레지스트리 `AutoAdminLogon` 전부 Windows 11 + Microsoft 계정 + PIN 조합에서 안 된다. Sysinternals만 된다.

#### 단계 4: WSL 재시작 시 자동 초기화

`/etc/wsl.conf`에:
```ini
[boot]
command=/home/dtsli/wsl-server-init.sh
```

PC 재부팅 후 WSL이 뜨면 init 스크립트 자동 실행.

### 3-3. 전체 자동 복구 체인

```
정전/윈도우 업데이트 → PC 전원 복구
    → BIOS AC Power Recovery = On  (자동 켜짐)
    → Sysinternals Autologon       (자동 로그인)
    → WSL 자동 시작 + sshd 시작   (wsl.conf boot.command)
    → Tailscale 자동 연결
    → 핸드폰에서 SSH 접속 가능
```

## 4장. SSH 안정성 — 끊기지 않는 연결

### 4-1. 왜 SSH가 끊기는가

실제 4중 복합 원인 (2026-03-21 발견):

| 원인 | 증상 | 해결 |
|------|------|------|
| SSH 자기루프 | CPU 38% 독식, 세션 드랍 | `ps aux \| grep ssh` → `kill` |
| tailscaled CPU 폭주 | 30% CPU, 연결 불안정 | `sudo kill [PID]` → 재시작 |
| `.wslconfig` 없음 | 7시간 WSL 자동종료 | `vmIdleTimeout=-1` |
| TTS 좀비 프로세스 | 메모리 점유 | `pkill -f script` |

### 4-2. autossh — 끊기면 자동 재접속

`pkg install autossh`

폰 `~/.bashrc`에 `pc` alias:
```bash
alias pc='autossh -M 0 \
  -o "ServerAliveInterval 30" \
  -o "ServerAliveCountMax 3" \
  -o "TCPKeepAlive yes" \
  -p 2222 dtsli@100.90.83.128 \
  -t "tmux attach -t claude-main || tmux new-session -s claude-main"'
```

WiFi ↔ LTE 전환해도 자동 재접속.

### 4-3. SSH 클라이언트 keepalive

`~/.ssh/config`:
```
Host pc-wsl
    HostName 100.90.83.128
    Port 2222
    User dtsli
    ServerAliveInterval 30
    ServerAliveCountMax 3
    TCPKeepAlive yes
    ConnectTimeout 10
```

### 4-4. tmux — 절대 규칙

> **Termux에서 tmux 쓰지 마라.**

올바른 패턴:
```
Termux 맨 bash (tmux 없이)
    → SSH
        → PC WSL2
            → claude-main tmux 세션에 attach
```

잘못된 패턴:
```
Termux tmux 안에서 SSH → PC tmux 중첩
→ 키 바인딩 충돌 + 화면 깨짐 + 세션 추적 불가
```

### 4-5. SSH 긴급 탈출법

연결이 완전히 먹통일 때:
```
~.
```
물결표(`~`) 누르고 바로 마침표(`.`) = SSH 강제 종료.

## 6장. 자동화 도구 — 어떤 도구를 언제 쓰는가

### 6-1. 도구 우선순위 (이 순서를 반드시 따를 것)

```
0순위: API / 터미널
    → 항상 먼저 확인. 되면 끝. 아래로 내려가지 않는다.

          ↓ API로 안 될 때만

1순위: Claude in Chrome (Chrome 확장)
    → GUI 클릭 필수 작업 (YouTube Studio, OAuth 등)
    → Claude가 브라우저 안에서 직접 보고 클릭

2순위: Playwright MCP
    → Claude가 브라우저 외부에서 직접 조작

3순위: CDP / 스크립트
    → 반복 배치. 사람 없이 야간 자동 실행
```

### 6-2. Termux에서 불가능한 것들

| 도구 | 이유 | 대안 |
|------|------|------|
| Playwright | Android 미지원 | GitHub Actions |
| Puppeteer | Android 미지원 | CDP 직접 연결 |
| OpenCV | aarch64 빌드 실패 | Pillow |
| scipy | 빌드 실패 | numpy |
| Docker | 커널 제약 | 불가 |

### 6-3. CDP — Termux에서 GUI 자동화하는 유일한 방법

```
Claude in Chrome (진짜 눈)  → PC 전용, 핸드폰 불가
Playwright                   → Termux 불가 (Android 미지원)
CDP (snap.js)                → ✅ Termux에서 돌아가는 점자
```

Playwright = 점자 (DOM 구조 손으로 더듬음)
Claude in Chrome = 눈 (화면 보고 판단)

## 7장. 지시하는 법 — 십장 모델

### 7-1. 십장 모델이란

> 십장은 못질 안 해도 된다.
> 어디에 뭘 지어야 하는지 알면 된다.

프로그래밍 언어를 배울 필요 없다.
Claude에게 **지시하는 법**만 배우면 된다.

기존 교육: 다리 만드는 법 가르침
이 교재: 다리 건너는 법 가르침

### 7-2. 나쁜 지시 vs 좋은 지시

**나쁜 지시 (모호함):**
```
"이거 고쳐줘"
"잘 안 돼"
"좀 더 좋게 만들어"
```

**좋은 지시 (맥락 + 증상 + 목표):**
```
"claude-tts.sh에서 tts_filter grep이
 Invalid collation character 에러 도배돼.
 grep 한글 정규식을 LC_ALL=C 안 타는 방식으로 고쳐.
 푸시해."
```

구성: **파일/위치** + **증상** + **원하는 결과** + **처리 방식**

### 7-3. Claude 역할 구분

| 역할 | 담당 | 하는 일 |
|------|------|---------|
| 설계/기획 | Opus | 전략, 아키텍처, 방향 |

Claude는 새 세션이면 이전 대화를 모른다.
그래서 지시할 때 맥락을 던져줘야 한다.

실제 예시:

```
 그리고 TTS가 폰으로 reverse tunnel 타고 가는 구조 확인해."
```

## 8장. 결과 검증 — 잘 됐는지 어떻게 아는가

### 8-1. 검증 3원칙

1. **눈으로 확인**: 화면에 오류 없이 원하는 결과가 보이는가
2. **재현 테스트**: 껐다 켜도 똑같이 동작하는가
3. **실패 시 롤백**: 안 되면 `git revert`로 되돌린다 (reset --hard 금지)

```bash
# SSH 루프 확인 (CPU 폭주 원인)
ps aux | grep ssh

# tailscaled 상태
systemctl status tailscaled

# wslconfig 확인
cat /mnt/c/Users/dtsli/.wslconfig

# 포트 점유 확인
lsof -i :9876

# tmux 세션 목록
tmux ls
```


## 9장. 삽질 사전 — 실전에서 만난 것들

### 9-1. tmux 화면이 깨졌을 때

**증상:** 점선 도배, 글자 겹침, 입력 안 됨

**해결:**
```bash
tmux kill-server       # 모든 tmux 세션 종료
tmux new -s work       # 새로 시작
```

살리고 싶은 세션 있으면:
```bash
tmux detach
tmux attach -t [세션명]
```

### 9-2. SSH 접속이 계속 끊길 때

**체크리스트:**
1. `ps aux | grep ssh` → SSH 루프 있는지 확인 → `kill [PID]`
2. `systemctl status tailscaled` → CPU 폭주 시 재시작
3. `.wslconfig` 에 `vmIdleTimeout=-1` 있는지 확인
4. Windows 절전모드 차단 됐는지 확인

### 9-3. Claude Code가 맥락을 모를 때

새 세션은 이전 대화를 모른다. 정상이다.
CLAUDE.md가 있는 레포에서 열면 기본 맥락은 읽는다.
나머지는 직접 던져줘야 한다.

### 9-4. 레지스트리 수정 권한 거부

**증상:** `Access to the registry key ... is denied`

**원인:** WSL에서 powershell.exe 호출은 관리자 권한이 없음

**해결:** Windows에서 PowerShell을 "관리자 권한으로 실행" → Claude Code 열기

### 9-5. 포트 충돌

**증상:** `connect_to localhost port 9876: failed`

**해결:**
```bash
lsof -i :9876          # 뭐가 잡고 있는지 확인
kill [PID]             # 죽이기
```

### 9-6. git pull이 안 될 때

**증상:** 로컬 수정과 원격이 충돌

**해결:**

## 부록 A. 새 PC 복구 순서

PC가 완전히 날아갔을 때:

```
1. Windows 설치
2. Windows Claude Code 설치 (PowerShell, 관리자 권한)
3. 로컬 PC 시스템 권한 MCP 붙이기
4. GitHub 토큰 주기
5. "나머지 다 세팅해" — 자동 업데이트 차단, SSH, Tailscale
6. WSL 설치 시키기
7. WSL 안에 Claude Code 설치 시키기
8. WSL 환경 세팅 (tmux, sshd, wslconfig, termux-bridge setup.sh)
```

Windows Claude Code가 산파 역할이다.
WSL을 낳아주고, WSL Claude Code를 낳아주고, 환경까지 세팅해준다.

## 부록 C. 자동화 경계 선언

> **termux-bridge produces evidence. Humans produce decisions.**

| 자동화 O | 사람 판단 O |
|---------|------------|
| 스크린샷, 링크체크, 에러수집 | broken link 분류 |
| JSON 리포트 생성, git 커밋 | 수정 구현, 우선순위 결정 |

의도적으로 자동화하지 않는 것:
- 자동 수정 (blast radius)
- GUI 클릭 자동화 (플랫폼 TOS)
- 로그인 페이지 체크 (credential 보안)
