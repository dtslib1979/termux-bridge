구글 드라이브 필요 없다 — 내 PC가 내 클라우드

■ 뭔 소리냐

집에 있는 PC를 24시간 서버로 돌려서, 핸드폰이랑 직접 파일 주고받는 구조를 만들었다. 구글 드라이브, iCloud 같은 클라우드 서비스 없이.

■ 어떻게 되는 건데

3개가 연결됨:
- Tailscale (VPN) → 폰이랑 PC를 암호화 터널로 연결. WiFi든 LTE든 어디서든.
- ADB (Android Debug Bridge) → PC에서 폰 파일을 직접 가져오거나 보낼 수 있음.
- Syncthing (자동 동기화) → 폰에서 사진 찍으면 자동으로 PC에 도착.

■ 구글 드라이브 vs 내 PC 서버

구글 드라이브:
- 용량: 15GB 무료, 초과시 월 구독
- 월 비용: 2TB 월 13,000원
- 보안: 구글 서버에 내 파일 저장됨
- 속도: 업로드/다운로드 거쳐야 함
- AI 학습: 구글이 내 파일 분석 가능
- 서비스 종료: 구글 맘대로

내 PC 서버:
- 용량: 하드 용량 = 내 클라우드 (20TB면 20TB)
- 월 비용: 전기세만 (하드는 한번 구매)
- 보안: 내 집에만 있음. 아무도 못 봄
- 속도: 기기끼리 직접 전송 (더 빠름)
- AI 학습: 불가능 (폐쇄망)
- 서비스 종료: 내 하드는 영원히 내 것

■ 보안이 왜 좋냐

3중 잠금 구조:
1. Tailscale — WireGuard 기반 VPN. 외부 접근 자체가 불가.
2. SSH — Tailscale 위에 또 한겹 암호화.
3. ADB — USB로 물리적 인증한 기기만 접근 허용. 네트워크로는 뚫을 수 없음 (Android 15 보안).

→ 누가 접근하려면 VPN 뚫고 + SSH 뚫고 + USB 물리 인증까지 해야 됨. 사실상 불가능.

■ 파일 전송 방식 비교

카톡 나에게 보내기: 압축됨, 깨짐, 느림, 자동화 불가
구글 드라이브: 15GB 제한, 보통 속도, 클라우드 경유
USB 케이블: 제한 없음, 빠름, 자동화 불가
Quick Share: 제한 없음, 빠름, 같은 WiFi만
ADB + Tailscale: 제한 없음, 빠름, 자동화 가능, 직접 연결, 어디서든

■ 실제 활용

- 사진 찍으면 → 자동으로 PC에 동기화
- "오늘 사진 가져와" → AI가 폰 사진 PC에 전송
- PC에서 편집한 파일 → 폰으로 자동 전송
- WiFi에서만 동기화 설정 가능 → LTE 데이터 과금 없음
- 3GB든 10GB든 용량 제한 없이 전송

■ 비용 정리

구글 드라이브 2TB: 연 156,000원 (매년 계속)
WD 20TB 외장하드: 약 350,000원 (한번만)
→ 2년이면 본전, 그 뒤로 공짜. 용량은 10배.

■ 전제 조건

- PC가 24시간 켜져있어야 함 (BIOS 설정으로 자동 부팅)
- Tailscale 설치 (무료)
- USB C to C 케이블로 한번만 폰↔PC 연결 (ADB 인증)
- 그 다음부터 영구 무선 연결

ADB 무선 연결 세팅 — USB 한번만 꽂으면 영구 무선

■ 필요한 것
- C to C 케이블 (충전 케이블이면 됨)
- PC (Windows, 24시간 켜져있는 거)
- 삼성 폰 (개발자 옵션 켜져있어야 함)

■ 사전 준비: 개발자 옵션 켜기
1. 설정 → 휴대전화 정보 → 소프트웨어 정보
2. "빌드번호" 7번 연타 → "개발자 모드 활성화됨" 뜸
3. 설정 → 개발자 옵션 → "USB 디버깅" ON

■ PC에 ADB 설치 (한번만)
1. 구글에서 "platform-tools" 검색 또는 아래 주소 접속
   https://dl.google.com/android/repository/platform-tools-latest-windows.zip
2. 다운받아서 C:\platform-tools\ 에 압축 해제
3. 폴더 안에 adb.exe 있으면 성공

■ USB 연결 + 무선 전환 (이게 핵심, 5분)
1. C to C 케이블로 폰 ↔ PC 연결
2. 폰에 "USB 디버깅 허용하시겠습니까?" 팝업 뜸
   → "이 컴퓨터에서 항상 허용" 체크 → 확인
3. PC에서 명령 프롬프트(cmd) 열고:

   cd C:\platform-tools
   adb devices

   → 기기 목록에 폰 나오면 연결 성공

4. 무선 모드로 전환:

   adb tcpip 5555

   → "restarting in TCP mode port: 5555" 뜨면 성공

5. USB 케이블 빼기 (이제 안 써도 됨)

6. 폰의 Tailscale IP 확인 (폰 Tailscale 앱에서 100.x.x.x)

7. PC에서:

   adb connect 100.x.x.x:5555

   → "connected to 100.x.x.x:5555" 뜨면 완료

■ 이후부터는
- PC 켤 때마다 adb connect 100.x.x.x:5555 한 줄이면 연결
- 자동화하려면 시작 스크립트에 넣으면 됨
- WiFi든 LTE든 Tailscale 타고 어디서든 연결됨

■ 되는 것들
- PC에서 폰 파일 가져오기: adb pull /sdcard/DCIM/사진.jpg C:\Users\내폴더\
- PC에서 폰에 파일 보내기: adb push 파일.pdf /sdcard/Download/
- 폰 화면 터치/앱 실행/설정 변경 전부 원격으로 가능
- 용량 제한 없음 (3GB든 10GB든)
- 클라우드 안 거침 (직접 연결)

■ 주의
- 폰 재부팅하면 adb tcpip 5555 다시 해야 함 (USB 다시 꽂거나 폰 Termux에서 가능)
- PC가 24시간 켜져있어야 어디서든 접근 가능
- Tailscale 양쪽 다 켜져있어야 함

■ 요약
USB 한번 꽂아서 "이 PC 믿어" 등록 → 케이블 빼기 → 영구 무선 연결
